# Installation

Get CX Linux running on your system in under 5 minutes. Choose your preferred installation method below.

## System Requirements

Before installing, ensure your system meets these minimum requirements:

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| RAM | 2 GB | 4 GB+ |
| Storage | 500 MB | 1 GB |
| Python | 3.9+ | 3.11+ |
| OS | Linux kernel 5.4+ | Latest LTS kernel |

## Quick Install (Recommended)

The fastest way to install CX Linux is using our install script:

```bash title="Install CX Linux"
curl -fsSL https://cxlinux.com/install.sh | bash
```

This script automatically:
- Detects your distribution and package manager
- Installs all required dependencies
- Configures your AI backend
- Sets up shell completions

## Package Manager Installation

<Tabs>
  <TabItem value="apt" label="Debian/Ubuntu" default>

```bash
# Add the CX Linux repository
sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg

curl -fsSL https://cxlinux.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/cx.gpg

echo "deb [signed-by=/usr/share/keyrings/cx.gpg] https://apt.cxlinux.com stable main" | \
  sudo tee /etc/apt/sources.list.d/cx.list

# Install CX Linux
sudo apt-get update
sudo apt-get install cx
```

  </TabItem>
  <TabItem value="dnf" label="Fedora/RHEL">

```bash
# Add the CX Linux repository
sudo dnf config-manager --add-repo https://rpm.cxlinux.com/cx.repo

# Install CX Linux
sudo dnf install cx
```

  </TabItem>
  <TabItem value="pacman" label="Arch Linux">

```bash
# Install from AUR using yay
yay -S cx-bin

# Or using paru
paru -S cx-bin
```

  </TabItem>
  <TabItem value="pip" label="pip (Universal)">

```bash
# Install via pip (requires Python 3.9+)
pip install cx-linux

# Or with pipx for isolated installation
pipx install cx-linux
```

  </TabItem>
</Tabs>

## Building from Source

For developers or those who want the latest features:

```bash
# Clone the repository
git clone https://github.com/cxlinux-ai/cx-core.git
cd cx-core

# Create virtual environment
python -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -e ".[dev]"

# Verify installation
cx --version
```

## Verify Installation

After installation, verify that CX Linux is working correctly:

```bash
# Check version
cx --version
# highlight-next-line
# Output: CX Linux v1.0.0

# Run a test command
cx "show system information"
```

## Post-Installation Setup

After installing, you'll need to configure your AI backend. See the [Configuration](/docs/getting-started/configuration) guide for details.

:::tip Shell Completions
Enable shell completions for a better experience:

```bash
# Bash
cx completion bash >> ~/.bashrc

# Zsh
cx completion zsh >> ~/.zshrc

# Fish
cx completion fish > ~/.config/fish/completions/cx.fish
```
:::

## Troubleshooting

### Common Issues

<details>
<summary>Command not found: cx</summary>

Ensure the installation directory is in your PATH:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

Add this line to your shell configuration file (`~/.bashrc`, `~/.zshrc`, etc.).

</details>

<details>
<summary>Permission denied during installation</summary>

If you encounter permission errors, try installing with `--user`:

```bash
pip install --user cx-linux
```

</details>

<details>
<summary>Python version too old</summary>

CX Linux requires Python 3.9 or newer. Check your version:

```bash
python3 --version
```

Use [pyenv](https://github.com/pyenv/pyenv) to manage multiple Python versions if needed.

</details>

## Uninstallation

To remove CX Linux from your system:

<Tabs>
  <TabItem value="apt" label="Debian/Ubuntu" default>

```bash
sudo apt-get remove cx
sudo rm /etc/apt/sources.list.d/cx.list
```

  </TabItem>
  <TabItem value="dnf" label="Fedora/RHEL">

```bash
sudo dnf remove cx
```

  </TabItem>
  <TabItem value="pip" label="pip">

```bash
pip uninstall cx-linux
```

  </TabItem>
</Tabs>
