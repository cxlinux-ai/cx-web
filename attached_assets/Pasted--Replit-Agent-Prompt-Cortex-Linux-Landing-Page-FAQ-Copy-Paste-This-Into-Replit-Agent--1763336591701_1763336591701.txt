Q1: Is Cortex safe? Won't it break my system?
A: Cortex is designed with security as the #1 priority. Here's how we protect your system:
   - Sandboxed Execution: Every command runs in an isolated Firejail container with no direct kernel access
   - Preview Before Execute: You see and approve every command before it runs
   - Instant Rollback: We create snapshots before major changes. Undo anything in seconds.
   - Command Validation: Dangerous commands (like rm -rf /) are blocked automatically
   - Audit Logging: Every action is logged for complete transparency
   
   Think of it like having a professional system administrator who shows you their plan before doing anything.

Q2: How do you prevent dangerous commands like 'rm -rf /'?
A: Multiple layers of protection:
   1. Blacklist: Destructive commands are blocked by default
   2. Dry-run mode: Test changes in simulation first
   3. User confirmation: System-wide changes require your explicit approval
   4. Severity scoring: AI classifies command risk level (safe/caution/danger)
   5. Rollback ready: Even approved commands can be undone
   
   You can review our security architecture on GitHub: [link]

Q3: What if the AI hallucinates or makes mistakes?
A: We've built multiple safeguards:
   - Command Validation: Every AI-generated command is validated against known patterns
   - Testing First: Commands are tested in sandbox before running on your system
   - Human-in-the-loop: You approve all critical operations
   - Learning System: When errors occur, Cortex learns and never repeats them
   - Confidence Scoring: Low-confidence suggestions require extra confirmation
   
   The AI suggests, you decide. Always.

Q4: Does Cortex have root access to my system?
A: No. Cortex runs with the same permissions as your user account. When root/sudo is needed:
   1. Cortex asks for your sudo password (just like any other program)
   2. Shows you exactly what will run with elevated permissions
   3. Waits for your approval
   4. You can always say no
   
   Cortex never stores your password or runs elevated commands without permission.

Q5: Can I review commands before they execute?
A: Yes, always. The workflow is:
   1. You: "Install Oracle optimized for my GPU"
   2. Cortex: Analyzes and shows you the plan:
      - Will install: CUDA 12.3, Oracle 23c, dependencies X, Y, Z
      - Will modify: /etc/oracle/config, ~/.bashrc
      - Estimated time: 4 minutes
   3. You: Approve or reject
   4. Cortex: Executes only after approval
   
   You can also enable "explain mode" where Cortex describes what each command does in plain English.
